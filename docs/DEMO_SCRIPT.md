# Сценарий демо-видео (≈ 3 минуты)

Видео по ТЗ оценивается по двум осям:

- **AI-окружение (50%)** - что и как настроено, чтобы агент закрывал процесс разработки.
- **Функционал и usability (50%)** - что итоговое приложение умеет.

Сценарий построен так, чтобы оба пункта раскрылись на одном проходе.

## Подготовка к записи

1. Открой два окна друг рядом с другом:
   - **Левая половина** - терминал с твоим Claude Code (открыт в `~/PetProjects/TestTask`).
   - **Правая половина** - Chrome / Safari в новом полноэкранном Space.
2. Убедись, что стек поднят: `make verify` или `docker compose ps` показывает 4 контейнера up.
3. Открой `http://localhost:3000/` заранее (чтобы Vite-кеш прогрелся).
4. Нажми `Cmd+Shift+5`, выбери **«Запись выделенного фрагмента»**, охвати оба окна, **«Записать»**.
5. Глубокий вдох - поехали.

## Структура (по таймкодам)

### 0:00–0:20 · «Это monorepo с AI-обвязкой» (вступление)

Покажи в терминале:
```bash
tree -L 2 -I 'node_modules|vendor|storage' | head -40
cat CLAUDE.md | head -30
```

**Что сказать:**

> «Тестовое задание: каталог авторских туров с семантическим поиском.
> Я собрал монорепу - Laravel, Vue с Vike, FastAPI для эмбеддингов
> и Postgres с pgvector. Главный артефакт AI-окружения - `CLAUDE.md`:
> он задаёт правила для агента, карту репозитория и чек-лист
> верификации перед сдачей».

### 0:20–0:45 · Sub-agents и slash-команды

```bash
ls .claude/agents/   # backend-expert, frontend-expert, ai-expert
ls .claude/commands/ # dev, verify, feature, add-tour, reseed
cat .claude/agents/backend-expert.md | head -15
```

**Что сказать:**

> «Три специализированных sub-agent: backend, frontend и AI-эксперт.
> У каждого свой scope и набор инструментов - Claude Code их вызывает,
> когда задача попадает в их зону. Плюс пять slash-команд:
> `/dev` поднимает стек, `/verify` гоняет тесты + typecheck + http-пробы,
> `/feature` оркеструет фичу через sub-agents,
> `/add-tour` создаёт тур через тот же LLM-путь что админка».

### 0:45–1:00 · Реальный запуск `/verify`

В Claude Code набери `/verify` (или прямо в терминале):
```bash
make verify
```

**Что сказать:**

> «Это smoke-гейт: тесты PHPUnit, vue-tsc и HTTP-пробы - всё параллельно.
> Зелёный сигнал - задача считается выполненной».

### 1:00–1:30 · Каталог + семантический поиск

Переключись на Chrome → `http://localhost:3000/`

1. Покажи карточки туров (6 штук, разные категории и сложности).
2. Кликни **«Гастро-тур на 3 дня»** в плашке подсказок поиска.
3. Покажи бейджи `match 45%` - каждая карточка имеет score.
4. Открой фильтры слева - выбери категорию «Горы» + слайдер цены.

**Что сказать:**

> «Каталог рендерится через SSR - Vike отдаёт готовый HTML,
> можно расшарить ссылку с фильтрами. Семантический поиск идёт
> через эмбеддинги: запрос превращается в 384-мерный вектор,
> pgvector сортирует туры по косинусной близости. Если сервис
> эмбеддингов выключить - есть фолбэк на ILIKE».

### 1:30–2:00 · Карточка тура

Открой `http://localhost:3000/tour/kamchatka-volcanoes` (или клик по карточке).

Покажи:
- Галерея с миниатюрами - переключай фото.
- Описание + 4 хайлайта.
- **Яндекс карта** с маршрутом и пронумерованными метками.
- Блок дат - выбери дату, цена меняется.

**Что сказать:**

> «Карточка содержит всё что просило ТЗ: фотоальбом, описание,
> длительность, категории, маршрут на Яндекс карте с метками точек
> и варианты дат + цена. Координаты редактируются в админке - карта
> подстраивается автоматически».

### 2:00–2:35 · Админка + LLM-генерация (главная фишка)

`http://localhost:8000/admin` → `admin@tours.local` / `password`.

1. Покажи список туров (с обложкой, категориями, счётчиком дат).
2. Кликни **«+ Создать»**.
3. В самом верху формы - **«Сгенерировать через LLM»**.
4. Введи в textarea: `Поход по Камчатке с восхождением на Авачинский вулкан, 5 дней, средний уровень`.
5. Жми **«Submit»** - поля заполняются за 2-3 секунды (название, описание,
   highlights, точки маршрута, даты).
6. Прокрути вниз, покажи как точки маршрута появились → можно сразу сохранить.

**Что сказать:**

> «Кнопка генерации идёт в Anthropic Messages API через прокси gngn.my -
> один интерфейс `TourGenerator`, два способа авторизации:
> прямой `x-api-key` или Bearer-токен прокси. Бэкенд сам выбирает
> заголовок. LLM возвращает JSON-черновик строго по схеме -
> я форсирую его через prefill `{` в assistant-сообщении.
> Админ дальше может править - поэтому я мягко валидирую,
> а не отбиваю кривой ответ ошибкой».

7. После сохранения покажи карточку нового тура → автоматически
   проиндексировался через `TourIndexer` (afterSave hook).

## Подсказки

- **Темп речи:** немного медленнее обычного - на видео всё кажется быстрее.
- **Курсор подсвечен:** в System Settings → Accessibility → Pointer
  включи «Increase pointer size» на время записи.
- **Если запутался** - стоп, не паникуй, mp4 можно урезать в QuickTime
  без перекодирования (`Edit → Trim`).
- **Размер файла:** ~30-50 МБ на 3 минуты ≈ влезет в репу. Если будет
  больше 100 МБ - заливай как Release asset:
  ```bash
  gh release create demo demo.mov --notes "Demo recording"
  ```

5. 60-80s - админка: «Сгенерировать через LLM» с одной фразой
6. 80-90s - `make verify` пробегает зелёным
